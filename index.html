Awesome Features

BAAS -Backend As AService

B.E.E is an intuitive open source backend as a service platform, which means that you dont need to develope a backend for your app. It takes care of the mandatory activities of a backend e.g CRUD (create, read, update, delete) operations, all you have to do is give it a request and it will do the hard work for you.

Multi-Tenant Architecture

The engine can serve multiple customers each storing its data in a separate database. The engine is smart enough to known how to connect to the right database from the incoming requests. The enginee also can track which customer has paid for what module and place access restrictions to unpaid modules of your application, just out of the box.

Request Bundling

This reduces the number of necessary HTTP requests needed to fullfill a specific operations by merging multiple requests into a single request object that is sent to the server at once and the server is happy to execute all operations, in turn saving you from doing lots of round trips.

Organic Requests

The client writes a structure that describes the data it needs from the server (NECTOR),
The backend enginee (BEE) interprets this structure into a series of highly optimised sql statements,pre operations and post operations. It then executes these and formats the data to be returned according to the structure of the request (HONEY),
finally the server sends back a this honey response to the client as json by default.

One API End Point

The only documentation that is shared among the team is the database design, there are no endpoints or versions of endpoints for your api. The only end point is the url to your server usually called the base url(HIVE location) e.g https://myawesomeapp.com. When a request arrives at the server, the engine (bee) checks the request body or query parameters for the structure (nector) of what you want from the database. The engine formats the response (honey) in the way you intend to use the data.

Plug and Play

B.E.E can be used on already existing applications to add new features or reduce the need to maintain ever changing api version end points and compatibility issues.
You can use it as an ORM internally to query for data from your db or make changes. New applications can quickly be developed using the enginee as a backend service.
There are many bee versions for different technology stacks e.g php & mysql, asp.net core mvp & mssql
